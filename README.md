# APR Patch Correctness Study

## Overview

This project investigates how developers evaluate the correctness of patches generated by Automated Program Repair (APR) systems, and how these evaluations are influenced by Large Language Models (LLMs).

Participants are asked to inspect buggy programs and their corresponding patched versions, and decide whether the patch is:

* **Correct** (generalizes beyond specific cases), or
* **Overfitting** (appears tailored to limited test cases and may not generalize),
* or **Not sure**.

The study is structured in multiple stages to measure how exposure to LLM outputs (decisions and reasoning) affects human judgment.

---

## Research Questions

This project is designed to explore the following:

1. **Alignment**
   How closely do LLM patch-correctness judgments match human judgments?

2. **Disagreement Patterns**
   In which cases do human and LLM judgments differ, and what types of patches are more prone to disagreement?

3. **Influence of Reasoning**
   Does LLM-provided reasoning influence human decisions, especially causing participants to revise their answers?

---

## Study Design

Each task follows a **three-stage evaluation process**:

### Stage 1 — Independent Judgment

Participants are shown:

* A buggy program
* A patched version

They classify the patch as:

* Correct
* Overfitting
* Not sure

---

### Stage 2 — LLM Decision Exposure

Participants are shown:

* The LLM’s classification (**Correct** or **Overfitting**)

They indicate:

* Agree
* Disagree
* Not sure

---

### Stage 3 — LLM Reasoning Exposure

Participants are shown:

* The LLM’s reasoning for its decision

They can:

* Keep their original answer
* Or revise it (Correct / Overfitting / Not sure)

## Directory Structure

```text
.
├── buggy_codes/          # Original buggy implementations
├── patched_codes/        # Corresponding patched implementations
├── tests/                # Test cases that shows the nature of the buggy programs
├── llm-queries/          # LLM reasoning logs (per algorithm)
└── llm-responses.csv     # Ground truth and LLM decisions
```

---

## LLM Responses File

`llm-responses.csv` contains:

```text
Algorithm,Truth,LLM Decision
```

* **Truth**: Ground truth label (Correct / Overfitting)
* **LLM Decision**: Model prediction

This file is used to:

* Compare human vs LLM judgments
* Identify disagreement cases
* Measure influence of LLM outputs

---

## Summary

This project provides a controlled setup to analyze:

* How developers judge APR patches
* Where LLMs agree or disagree with humans
* Whether LLM reasoning changes human decisions

The goal is not to assume correctness of either humans or LLMs, but to understand **their interaction**.
